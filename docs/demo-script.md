# Demo Script

The final demo should show OpsFlow as an enterprise operations workflow application, not a generic ticket tracker.

## Planned Demo Flow

1. Sign in as a manager.
2. Review dashboard metrics.
3. Open the case queue and identify overdue or high-priority work.
4. Inspect a case detail page with SLA, notes, assignment, status, approval state, and audit timeline.
5. Reassign or progress a case according to role permissions.
6. Submit or decide a manager approval.
7. Return to the dashboard and show metrics updated from SQL-backed data.

## PR-01 Status

No application demo workflow exists yet. PR-01 seeds the database with realistic synthetic cases, overdue-ready work, pending approval samples, notes, histories, and audit logs so later PRs can support the final demo flow.
